# Insight Forge — UX / UI 草图与设计理念

> 版本：v0.1  
> 日期：2025-11-18  
> 作者：AI Product Manager

本文件用于沉淀 AI 知识管理系统（AI KnowledgeOS）中 **Insight Forge** 模块的体验规范、界面草图与设计理念，方便后续迭代、协作与作品集展示。

---

## 1. 设计目标

1. 让上传后的文档在 **30 秒内**转化为结构化、可信赖、可个性化的知识资产。  
2. 提供跨 Persona 的知识透视，体现“为你定制的第二大脑”。  
3. 保证 AI 生成内容的 **追溯性、可编辑性与协作性**。

---

## 2. 页面基线结构（Wireframe 概述）

```
┌ Hero Zone：文档信息 + 处理状态
├ AI Knowledge Cards（横向卡片流）
└ Insight Canvas（三列）
    ├ Summary & Key Points
    ├ Perspective Comparison
    └ AI Insights & Next Steps + Persona Switcher
└ Reference Strip（原文导航）
```

### 模块说明

| 区域 | 说明 | 设计要点 |
|------|------|----------|
| Hero Zone | 展示文档标题、来源、标签、生成进度 | 解决“上传完系统在干嘛”的焦虑 |
| AI Knowledge Cards | 将文档拆成原子知识卡 | 类型标签、引用来源、收藏/标注等操作 |
| Insight Canvas | 三列布局：摘要、观点对比、AI 下一步 | 兼顾宏观 TL;DR 与可操作洞察 |
| Persona Switcher | 切换不同角色视角 | 默认 + 团队模板 + 对比模式 |
| Reference Strip | 原文引用、章节导航 | 卡片/观点点击后同步定位 |

## 3. 视觉与交互基调

### 3.1 视觉语言

- **色彩**：深色背景 + 高对比模块，突出“专业分析工具”质感。  
- **卡片**：圆角 + 阴影层次，区分不同知识类别。  
- **字体**：标题使用半粗体，正文保持易读性。  
- **图标**：统一 16px 线性图标，覆盖收藏、复制、引用等操作。

### 3.2 交互模式

- **状态反馈**：上传完成 → 进度条 → 完成 toast。  
- **引用追溯**：卡片 hover 显示原文引用，参考高亮。  
- **Persona 切换**：即时渲染 + 差异高亮（绿/橙）。  
- **协作钩子**：卡片、摘要、观点都支持批注 / 👍👎 / “报告错误”。  
- **可编辑性**：AI 版本 vs 手动版本清晰区分，允许回滚。

## 4. 组件拆解

### 4.1 卡片组件

```
[类型] 标题（截断）
摘要一句
引用来源 (Pg.3, para 2)
[收藏] [复制] [批注] [更多]
```
- 状态：Skeleton / 加载失败（retry）  
- 批量操作：多选后“批量收藏/导出”

### 4.2 摘要组件

- Tabs：`TL;DR / 深度摘要 / 数据要点`
- 每段支持：复制、编辑、引用定位
- 深度摘要可展开 markdown 块

### 4.3 观点对比

- 列：`支持观点 / 反对观点 / 中立补充`
- 每条观点带权重或信心值 + 引用
- 可切换“列表”与“关系图”视图

### 4.4 Persona 面板

- Persona 选择器 + 管理按钮
- 当前 persona 的重点洞察、行动建议
- “差异高亮”显示与默认视角不同之处

### 4.5 Reference Strip

- 缩略卡 + 章节标题
- 点击即展开下方原文预览 / 跳转至全屏阅读

---

## 5. 设计理念总结

1. **先安抚、再呈现**：Hero Zone 明确告知系统状态，降低等待焦虑。  
2. **从点到面**：卡片→摘要→观点，帮助用户迅速建立全局认知。  
3. **为不同角色服务**：Persona 是一级入口，让智能知识真正“懂你”。  
4. **可信与可追溯**：引用、差异、高亮、反馈机制确保用户信任 AI 输出。  
5. **沉淀即资产**：每个卡片/摘要都可收藏、分享、复用，支撑整个 KnowledgeOS。
