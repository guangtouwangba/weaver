# AI KnowledgeOS — 页面架构与衔接

> 版本：v0.1  
> 日期：2025-11-18  
> 作者：AI Product Manager

本文件记录 AI KnowledgeOS（Capture → Insight → Action → Graph → Observability）的主要页面、职责、前后衔接与注意力管理策略，供产品、设计、前端协同参考。

---

## 1. Capture Hub（知识采集）

| 页面 | 职责 | 前后衔接 | 注意力管理 |
|------|------|----------|------------|
| **Workspace Dashboard** | 汇总“待处理文档 / 新任务 / Insight 推送”，作为系统入口 | 登录后默认页；可跳转 Capture Hub、Insight Forge、监控页 | 仅展示 Top 3 待处理项 + CTA（继续处理 / 去 Insight），减少干扰 |
| **多源上传 / Intake** | 选择来源（文档、网页、对话、邮箱等），配置主题标签 | 上传完成即时跳转“任务队列”；失败返回并提示重试 | 分步表单 + 顶部进度，确保用户清楚当前步骤 |
| **任务队列 / Processing 状态** | 追踪解析进度、错误、质量评分 | “已完成”→ CTA 前往 Insight Forge；“需人工校验”→ 内容审阅页 | 列表按优先级排序；高亮失败项；右侧固定帮助/FAQ |
| **内容审阅 / QA** | 人工验证 AI 解析的概念/标签 | 确认后推入 Insight Forge；拒绝则回任务队列 | 中央呈现 AI 结果，两侧为建议与人工编辑，减少视线跳转 |

---

## 2. Insight Forge（知识内化）

| 页面 | 职责 | 前后衔接 | 注意力管理 |
|------|------|----------|------------|
| **Insight Forge 主页面** | 将单文档转化为知识卡片、摘要、观点对比、Persona 版本 | 上游：任务队列；下游：卡片库、Action Engine、Graph Studio | 布局自上而下：状态 → 卡片 → 深度 Insight → 原文；Persona 切换固定在右上，强调个性化 |
| **Persona 模板管理** | 定义、编辑、对比 persona 权重 | 从 Insight Forge“管理 persona”进入；保存后影响后续生成 | 左侧列表 + 右侧表单，顶部提示“当前 Insight 使用的模板” |
| **知识卡片库** | 浏览、搜索、收藏所有卡片，沉淀知识资产 | Insight Forge 卡片 → “查看全部”进入；可推送到 Action Playbook | 表格 + 过滤器 + 预览；默认按更新时间排序；顶部显示卡片数量与主题构成 |

---

## 3. Action Engine（知识提取 / 应用）

| 页面 | 职责 | 前后衔接 | 注意力管理 |
|------|------|----------|------------|
| **场景化问答控制台** | 运行 RAG + 场景模板，输出带引用的回答 | Dashboard 快捷入口或 Insight Forge“发送到 Action” | 左侧场景设置，中间问答，右侧引用/评估；输出后自动 focus 引用块，强调可信度 |
| **Playbook / Workflow Builder** | 把卡片与摘要组合成任务流程或模板化输出 | 来自问答结果或卡片库“添加到 Playbook”；下游 → 导出/协作 | 时间线式布局，每步引用卡片，顶部提示适用 persona/场景 |
| **回答历史 / 引用追踪** | 查看历史请求、评估得分、引用链路 | Action 控制台“历史”页；也可由评估报告跳转 | 表格列出问题、得分、引用数；点击展开引用详情与再生按钮 |

---

## 4. Graph Studio（知识链接）

| 页面 | 职责 | 前后衔接 | 注意力管理 |
|------|------|----------|------------|
| **Graph Explorer** | 可视化知识网络、AI 导航洞察 | Insight Forge 顶部“查看图谱”或 Dashboard 推荐进入 | 图为中心，边缘控件轻量；AI 建议面板延迟出现，避免初始过载 |
| **Cross-Topic Linker** | 发现不同主题之间的桥接节点 | Graph Explorer“合并主题”CTA 或 Dashboard 推荐 | 分屏展示两主题概览，中央突出桥接节点；顶部状态条提示计算进度 |
| **Share / Embed 设置** | 生成只读链接、嵌入代码、查看访问统计 | Graph Explorer“分享”按钮进入；外部访问为只读视图 | Stepper 弹窗：范围 → 权限 → 链接/嵌入，单次专注一个配置项 |

## 5. Observability & Evaluation

| 页面 | 职责 | 前后衔接 | 注意力管理 |
|------|------|----------|------------|
| **Ops 监控仪表盘** | 监控延迟、错误、成本、缓存命中率 | Ops 角色首页或侧栏“监控”进入；告警链接至 Trace | KPI 卡片置顶，折线图居中，右侧列显示告警/快捷入口 |
| **Evaluation 报告** | 查看 runtime evaluation 得分、趋势与建议 | Action Engine 输出或 Dashboard “质量报告”进入 | 顶部显示本期总分 + 样本，列表按“问题→得分→引用→改进”顺序，引导关注重点 |
| **Log / Trace Viewer** | 调试具体请求 | 监控告警或开发工具栏进入 | 顶部快速过滤（Trace ID / 用户 / 时间），中间时间轴，底部原始日志，视觉路径清晰 |

## 6. 用户旅程示例

1. **知识工作者**：Dashboard → 上传文档 → Insight Forge（调整 persona）→ 选卡片加入 Playbook → Graph Explorer 分享  
   - 注意力脉络：状态确认 → 知识提炼 → 应用输出 → 结构展示

2. **运营 / Ops**：Dashboard → 监控告警 → Trace Viewer → 回 Insight Forge 修正卡片 → 重新触发评估  
   - 注意力脉络：指标波动 → 根因 → 内容修正 → 复验证

3. **协作/分享**：Insight Forge 生成 → Graph 分享 → 外部只读页 → 读者复制到个人库  
   - 注意力脉络：洞察生成 → 可视化呈现 → 权限控制 → 资产沉淀

---

## 7. 注意力管理原则

1. **阶段化呈现**：遵循“状态反馈 → 内容感知 → 深度操作”的节奏，减少信息焦虑。  
2. **逐步披露**：复杂配置采用 Stepper 或可折叠面板，确保用户只关注当前任务。  
3. **显式衔接 CTA**：每个页面顶部或右上角有“下一步”按钮，防止迷路。  
4. **角色化入口**：Dashboard 根据 persona 显示定制化模块入口，降低认知负担。  
5. **统一反馈**：Toast + 状态栏 + 历史记录三位一体，保证操作结果可感知、可追溯。
