# Research Agent RAG - Minimal Functional Specification

## 概述

本系统提供一个最小可用的检索增强生成（RAG）服务，聚焦于文档入库、语义检索与基础问答三大核心场景，支持单机部署和后续扩展。

## 目标用户

- 希望构建内部知识库的团队或个人
- 需要快速验证 RAG 工作流可行性的研发人员

## 核心功能

1. **文件导入**
   - 支持至少一种常见文本资料格式（例如 PDF 或 Markdown）。
   - 将原始文件转换为系统统一的 Document 结构，记录元信息（标题、来源、上传时间）。

2. **文档处理与向量化**
   - 对文档内容执行分块、清洗及嵌入生成。
   - 向量写入默认的向量数据库（保留一种实现，如 Weaviate或等价替代）。

3. **语义检索 API**
   - 提供 REST 接口按查询语句返回最相关的文档片段及其来源信息。
   - 返回结果需包含匹配得分或置信度，便于前端排序展示。

4. **基础问答接口**
   - 在检索结果基础上返回结构化回答，可先输出命中的片段摘要。
   - 为后续接入生成式模型预留扩展点。

5. **基础观测与日志**
   - 记录最小限度的结构化日志，便于排查问题。

## 非目标

- 不强制提供多模型切换、工作流编排或复杂权限体系。
- 不包含完整前端界面，仅关注 API 层与后台处理链路。

## 后续扩展建议

- 引入多格式文件解析与内容抽取增强模块。
- 结合混合检索（向量 + BM25）与重排序以提升命中率。
- 加入异步任务队列处理批量文档、摘要生成等高级能力。
