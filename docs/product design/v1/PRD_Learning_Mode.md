## 1. 问题描述

### 1.1 用户痛点

#### 痛点一：单一交互模式无法满足不同学习需求

**场景描述**：
- 用户A在快速查阅资料时，希望AI直接给出答案，节省时间
- 用户B在理解复杂概念时，希望AI能引导思考，帮助深入理解
- 用户C在研究学术问题时，希望通过提问自己发现答案，培养批判性思维

**当前问题**：
- 现有系统只有一种"直接回答"模式
- 无法根据用户的学习目标调整AI的互动方式
- 用户无法选择适合自己的学习深度

#### 痛点二：被动接受答案，缺乏主动思考

**场景描述**：
- 用户提问"什么是RAG？"
- AI直接给出完整答案
- 用户看完答案，但可能没有真正理解，只是"知道"了答案

**当前问题**：
- 用户被动接受信息，缺乏主动思考过程
- 难以形成深度理解和长期记忆
- 无法培养批判性思维和独立思考能力

#### 痛点三：复杂概念难以真正理解

**场景描述**：
- 用户学习"机器学习中的过拟合"概念
- AI给出定义和解释
- 用户仍然困惑：为什么会出现过拟合？如何避免？

**当前问题**：
- 直接答案可能无法触及用户真正的困惑点
- 缺乏引导用户自己发现问题的机制
- 无法帮助用户建立知识之间的联系

### 1.2 用户心理状态

**使用AI助手时的心理状态**：
1. **快速查阅时**：焦虑、急于获取答案 → 希望直接、快速
2. **学习新概念时**：好奇、想要理解 → 希望有引导、有思考
3. **深入研究时**：探索、想要深度理解 → 希望主动思考、自己发现

**当前系统的心理体验**：
- 快速查阅：满足（直接答案）
- 学习新概念：部分满足（有答案，但缺乏引导）
- 深入研究：不满足（被动接受，缺乏思考过程）

## 2. 解决方案：三级学习模式

### 2.1 核心概念

**学习模式** = 不同的AI互动方式，帮助用户以不同深度理解知识

**设计理念**：
- 尊重用户的学习节奏和偏好
- 提供渐进式的学习深度选择
- 不强制，让用户自主选择

### 2.2 三级模式体系

#### Level 1: 快速回答 (Quick Answer)

**定位**：效率优先，直接获取答案

**适用场景**：
- 事实查询："2023年Q4的营收是多少？"
- 定义查询："什么是API？"
- 数据查询："文档中提到几个关键指标？"

**用户心理**：
- 状态：时间紧迫，需要快速信息
- 期望：直接、准确、快速
- 满足感：立即获得答案

**AI行为**：
- 直接给出完整答案
- 引用文档来源
- 简洁明了，不展开

#### Level 2: 引导思考 (Guided Thinking)

**定位**：答案+思考，帮助理解

**适用场景**：
- 概念理解："如何理解RAG的工作原理？"
- 原理探索："为什么需要向量检索？"
- 关系理解："A和B之间有什么关系？"

**用户心理**：
- 状态：想要理解，但不知道从何思考
- 期望：有答案，但希望有思考方向
- 满足感：既获得答案，又知道如何思考

**AI行为**：
- 先给出答案
- 然后提出1-2个引导性问题
- 帮助用户深入思考

**示例对话**：
```
用户: 什么是RAG？

AI: [给出答案]
    RAG是检索增强生成...

    💡 思考一下：
    • 为什么需要RAG？它解决了什么问题？
    • 如果没有RAG，AI回答会有什么局限？
```

#### Level 3: 深度探索 (Deep Exploration)

**定位**：通过提问引导，自己发现答案

**适用场景**：
- 复杂概念："如何理解机器学习中的过拟合？"
- 批判性思维："这个结论是否总是成立？"
- 深度研究："如何设计一个RAG系统？"

**用户心理**：
- 状态：想要深度理解，愿意花时间思考
- 期望：通过思考自己发现答案
- 满足感：理解更深刻，记忆更牢固

**AI行为**：
- 不直接给答案
- 通过提问引导用户思考
- 根据用户回答继续提问
- 最终帮助用户自己得出结论

**示例对话**：
```
用户: 什么是RAG？

AI: 🤔 在回答之前，我想先了解：
    你理解的"检索"是什么意思？
    你希望AI如何帮助你处理文档？

用户: 检索就是从文档中找到相关内容...

AI: 很好！那么，如果AI直接回答，不检索文档，会有什么问题？

用户: 可能会回答错误，或者没有依据...

AI: 对！这就是RAG要解决的问题。现在你能总结一下RAG是什么吗？
```

## 3. 交互设计

### 3.1 模式选择器

**位置**：Assistant Panel 头部，在"Assistant"标题下方

**视觉设计**：
```
┌─────────────────────────────────────────┐
│ 🟢 Assistant                    [−]    │
├─────────────────────────────────────────┤
│ 学习模式: [⚡ 快速回答 ▼]              │  ← 新增
│                                         │
│ [对话历史区域]                          │
└─────────────────────────────────────────┘
```

**交互流程**：
1. 点击下拉箭头 → 展开模式选择菜单
2. 选择模式 → 立即切换，显示简短说明
3. 模式切换后 → 当前对话继续，新问题使用新模式

**视觉反馈**：
- 当前模式：显示图标 + 名称，蓝色高亮
- 切换时：显示简短说明卡片（2秒后自动消失）
- 模式标识：在对话中显示当前模式图标

### 3.4 智能推荐提示

**触发条件**：
- 检测到适合深度探索的问题（如："如何理解..."、"为什么..."）
- 用户当前使用 Level 1 或 Level 2

**提示设计**：
```
┌─────────────────────────────────────────┐
│ 💡 这个问题适合用"深度探索"模式          │
```

**交互规则**：
- 每个问题类型只提示一次
- 用户点击"不再提示" → 记住偏好，不再显示
- 用户点击"试试深度探索" → 切换到 Level 3，开始对话

## 4. 用户心理曲线

### 4.1 首次使用流程

**阶段一：发现（0-30秒）**
- **心理状态**：好奇、探索
- **行为**：看到"学习模式"选择器，点击查看
- **情绪**：轻微兴奋，想知道是什么
- **设计要点**：清晰的说明，不复杂

**阶段二：理解（30秒-2分钟）**
- **心理状态**：理解、评估
- **行为**：阅读模式说明，理解差异
- **情绪**：理性思考，判断是否适合自己
- **设计要点**：简洁的描述，明确的适用场景

**阶段三：尝试（2-5分钟）**
- **心理状态**：尝试、验证
- **行为**：选择一种模式，提问测试
- **情绪**：期待、验证是否符合预期
- **设计要点**：立即反馈，符合预期

**阶段四：适应（5分钟+）**
- **心理状态**：适应、习惯
- **行为**：根据场景切换模式
- **情绪**：舒适、自然
- **设计要点**：流畅切换，无阻碍

### 4.2 不同模式的心理体验

#### Level 1 (快速回答)

**心理曲线**：
```
焦虑(提问) → 期待(等待) → 满足(获得答案) → 平静(完成)
```

**情绪变化**：
- 提问时：轻微焦虑（需要快速答案）
- 等待时：期待（希望准确）
- 获得答案：满足（问题解决）
- 完成后：平静（任务完成）

**设计要点**：
- 快速响应（减少等待焦虑）
- 准确答案（满足期待）
- 简洁明了（不增加认知负担）

#### Level 2 (引导思考)

**心理曲线**：
```
好奇(提问) → 期待(等待) → 满足(获得答案) → 思考(引导问题) → 理解(深入)
```

**情绪变化**：
- 提问时：好奇（想要理解）
- 等待时：期待（希望有深度）
- 获得答案：满足（有答案了）
- 看到引导问题：思考（想要深入）
- 思考后：理解（更深入了）

**设计要点**：
- 先给答案（满足基本需求）
- 引导问题要精准（触发思考）
- 不强制（用户可以选择不思考）

#### Level 3 (深度探索)

**心理曲线**：
```
探索(提问) → 困惑(被提问) → 思考(回答) → 继续探索(被继续提问) → 发现(自己得出结论) → 成就感(理解深刻)
```

**情绪变化**：
- 提问时：探索（想要深度理解）
- 被提问时：困惑（为什么问我？）
- 思考回答：专注（认真思考）
- 继续被提问：探索（继续思考）
- 自己得出结论：成就感（我理解了！）
- 完成后：满足（理解深刻）

**设计要点**：
- 温和的提问（不让人感到被审问）
- 清晰的引导（知道思考方向）
- 退出机制（可以随时退出）
- 及时肯定（用户回答得好时给予肯定）

### 4.3 模式切换的心理体验

**从 Level 1 → Level 2**：
- **心理**：从"快速获取"到"想要理解"
- **情绪**：从效率导向到学习导向
- **设计要点**：平滑过渡，不突兀

**从 Level 2 → Level 3**：
- **心理**：从"有引导"到"完全自主"
- **情绪**：从舒适区到挑战区
- **设计要点**：明确说明差异，让用户做好准备

**从 Level 3 → Level 1**：
- **心理**：从"深度思考"到"快速获取"
- **情绪**：从专注到放松
- **设计要点**：提供"直接给我答案"选项，尊重用户选择

## 5. 问题解决验证

### 5.1 痛点一：单一交互模式 → 解决

**解决方案**：三级学习模式，用户自主选择

**验证指标**：
- 用户在不同场景下选择不同模式
- 用户满意度提升（针对不同学习需求）
- 模式使用分布（快速回答 > 引导思考 > 深度探索）

### 5.2 痛点二：被动接受答案 → 解决

**解决方案**：Level 2 和 Level 3 引导主动思考

**验证指标**：
- Level 2 用户会思考引导问题
- Level 3 用户会主动回答AI的提问
- 用户反馈"理解更深刻"

### 5.3 痛点三：复杂概念难以理解 → 解决

**解决方案**：Level 3 通过提问引导用户自己发现

**验证指标**：
- 复杂概念问题选择 Level 3 的比例
- Level 3 对话轮次（越多说明思考越深入）
- 用户最终自己得出结论的比例

## 6. 设计原则

### 6.1 人本主义设计

**自主选择**：
- 默认 Level 1，不强制切换
- 所有模式平等，无优劣暗示
- 可随时切换，无限制

**渐进式引导**：
- 首次使用：欢迎提示（可关闭）
- 使用中：智能推荐（可关闭）
- 不打断用户流程

**清晰反馈**：
- 模式切换有视觉反馈
- 对话中显示当前模式标识
- 说明简洁，避免术语

**容错设计**：
- Level 3 中可随时退出
- 提供"直接给答案"选项
- 支持模式间无缝切换

### 6.2 心理安全

**不强迫**：
- 不强制用户使用深度模式
- 不暗示"深度模式更好"
- 尊重用户的学习节奏

**可退出**：
- Level 3 中始终提供退出选项
- 退出不损失已获得的信息
- 可以随时切换回其他模式

**及时肯定**：
- 用户回答得好时给予肯定
- 用户思考深入时给予鼓励
- 营造安全的学习环境

## 7. 成功指标

### 7.1 使用指标

- **模式分布**：各模式的使用比例
- **切换频率**：用户在不同模式间切换的频率
- **深度探索完成率**：Level 3 对话完成的比例

### 7.2 体验指标

- **用户满意度**：针对不同模式的满意度评分
- **理解深度**：用户反馈"理解更深刻"的比例
- **推荐意愿**：用户愿意推荐给朋友的比例

### 7.3 学习效果指标

- **记忆持久性**：使用不同模式后，知识记忆的持久性
- **应用能力**：用户能否将学到的知识应用到新场景
- **批判性思维**：使用 Level 3 后，用户批判性思维的提升

## 8. 后续优化方向

### 8.1 个性化推荐

- 根据用户历史使用习惯，智能推荐模式
- 学习用户在不同问题类型下的模式偏好

### 8.2 模式组合

- 支持"先快速回答，再深度探索"的组合模式
- 根据问题复杂度自动推荐模式

### 8.3 学习路径

- 为复杂主题设计学习路径
- 从 Level 1 到 Level 3 的渐进式学习
